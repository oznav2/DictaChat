# DictaLM 3.0
# BricksLLM  צ'אט בינה מלאכותית ישראלי על תשתית
**גירסה 1.0.0**

**שלום חברים!** 👋
פרויקט DictaChat זה נועד להפוך את השימוש במנוע הבינה המלאכותית הישראלי הכי מתקדם, **DictaLM-3.0**, לפשוט חינמי שלם ונגיש לכולם.
השתדלתי לבנות עבורכם מערכת "הכל-כלול" המאפשרת להריץ את המודל המדהים והפתוח מבית היוצר של דיקטא כשיתוף פעולה פורה שלה עם מפא״ת ולכזה שיהיה זמין אצלכם במחשב בבית בעסק או בארגון, כשהדגש בישום הוא על אבטחת המידע ושמירת פרטיות מלאה, כשהכל עטוף עם ממשק צ'אט נוח ומוכן לשימוש.

חיברתי אליו עשרות תכונות ופונקציות מתקדמות באופן שתקבלו חיבוריות מיידית לאינספור מקורות מידע מהרשת וסביבת העבודה בה המערכת מותקנת

## 🌟 למה להשתמש בזה?

* 🧠 קודם כל השתדלתי לפתור בעיה בתבנית השיחה של המודל שבכל יתר הפלטפורמות גורם למודל לספק תשובות בלתי נגמרות
*   🛡️ **פרטיות מוחלטת**: הכל רץ על המחשב שלכם. שום מילה לא יוצאת לענן או לשרתים חיצוניים.
*   💬 **ממשק צ'אט מוכן**: כולל מסך שיחה (כמו ChatGPT) שתומך בעברית ומציג את "תהליך החשיבה" של המודל.
*   🚀 **התקנה קלה**: במקום להסתבך עם עשרות הגדרות, פקודה אחת מריצה את הכל.
*   🚀 **התממשקות מהירה למודל**: בניתי את ממשק התקשורת עם המודל על בסיס שפת go lang
*   🧠 **ניהול חכם**: המערכת דואגת לכל "האינסטלציה" הטכנית מאחורי הקלעים באופן אוטומטי, וכוללת יכולות להגדיר תכונות מתקדמות. לא חובה לשנות כלום. רק אם רוצים:
    *   ⏳ **מגבלות שימוש**: הגדרת מכסות שימוש (Rate Limits) למשתמשים שונים לפי רמות תמחור.
    *   📊 **מעקב ובקרה**: ניטור צריכת משאבים ברמת משתמש או ארגון.
    *   🛡️ **אבטחת מידע**: חסימה או צנזור אוטומטי של מידע רגיש (PII) בבקשות.
    *   🔄 **אמינות**: מנגנוני גיבוי (Failover), ניסיונות חוזרים (Retries) ומטמון (Caching) לשיפור היציבות.
    *   🔑 **ניהול מפתחות**: הפצת מפתחות API עם מגבלות קצב ועלות למפתחים, סביבות ייצור, או סטודנטים.

## 🧠 מערכת הכלים החכמה - Enterprise Tool Intelligence

בניתי מערכת מתקדמת שגורמת למודל לפעול בסביבה מאורגנת, מבוקרת מודעת ומוכוונת כלים ונתונים בתצורה רב שלבית. המשמעות היא שהתכוונתי לאפשר לכם להנות ממודל שיכול להיות הרבה יותר חכם מאובטח ומבוקר באופן שהוא עושה את הדברים והכל תוך שימוש בכלים רבים שצירפתי לארסנל שלו. במקום שהמודל יתבלבל או יכשל, המערכת דואגת לכל הלוגיסטיקה מאחורי הקלעים:

### מה המערכת יודעת לעשות?

* 🎯 **בחירה חכמה של כלים**: כששואלים שאלה בעברית, המערכת מזהה את הכוונה ובוחרת את הכלי המתאים ביותר. למשל, אם שואלים על נתונים ממשלתיים - היא תפנה קודם למאגר data.gov.il. 
המערכת יוצאת מכלל הנחה שהמשתמש ישראלי שמתגורר בישראל ורוצה לקבל תשובות בעברית אלא אם ביקש מפורשות אחרת

* 🔄 **מנגנון Fallback אוטומטי**: אם כלי אחד נכשל או לא מחזיר תוצאות, המערכת מנסה אוטומטית כלים חלופיים לפי סדר עדיפויות. המשתמש לא תמיד רואה את זה - פשוט מקבל תשובה

* 📝 **תיקון פרמטרים אוטומטי**: המודל לפעמים עלוי לשלוח פרמטרים לא נכונים לכלים. המערכת מתקנת את זה אוטומטית - למשל, אם המודל שלח `query` במקום `messages`, המערכת יודעת להמיר. ואינסוף מאמץ אפשר להשקיע במיטוב המערכת אף מעבר למה שכבר עשיתי

* ⏱️ **ניהול זמנים חכם**: כלי מחקר מעמיק מקבל 5 דקות לרוץ ברקע, חיפוש מהיר מקבל דקה. ככה לא נתקעים על timeouts מיותרים

* 💬 **הודעות שגיאה בעברית**: במקום שגיאות טכניות מפחידות, המשתמש מקבל הסבר ברור מה קרה ומה אפשר לעשות

* 🤖 **המודל מכיר את היכולות שלו**: אם תשאלו "מה אתה יכול לעשות?" או "אילו כלים יש לך?", המודל ידע לענות ואפילו יציע כלים רלוונטיים תוך כדי השיחה. אשרי המאמין ואשרי המקווה לטוב תמיד.

## 🏛️ חיבור למאגרי המידע הממשלתיים (DataGov)

הוספתי חיבור מקיף לכל מאגרי הנתונים הפתוחים של ממשלת ישראל. עכשיו אפשר לשאול שאלות כמו:
- "כמה רכבים חשמליים יש בישראל?"
- "תן לי רשימת בתי חולים עם מספרי טלפון"
- "מה התקציב של משרד החינוך?"

### מה מיוחד בחיבור הזה?

בניגוד לתשובות של מודל בינה שמאומן על המידע עליו אומן, המערכת מציגה תשובות מעודכנות, מוסמכות לפי הנתונים האחרונים ועם אסמכתאות למקור המידע.

כל המידע הצבור במאגרי הידע הממשלתי מתוחזקים ומתעדכנים באופן אוטומטי, חלקם גם מוזנים במידע חדש בכל רבע שעה.

* 📊 **1,190 מאגרי נתונים מוכנים מראש** - כל מאגר ממשלתי כבר מחווט למערכת עם מטא-דאטה מפורט על השדות שלו

* 🔍 **הבנת עברית מתקדמת**: המערכת מבינה שינויים מורפולוגיים - "ברכבים", "לרכבים", "רכבים" - הכל מתורגם נכון

* 📍 **הפרדה בין מה לאיפה**: כששואלים "תן לי רשימה של בתי חולים בירושלים", המערכת מבינה ש"בתי חולים" זה הנושא ו"ירושלים" זה המיקום

* 🔢 **חישוב סיכומים אוטומטי**: שאלות "כמה" מקבלות תשובה מספרית ישירה, לא רשימה ארוכה של שורות

* 🌐 **22 תחומים עם תרגום דו-כיווני**: משפטים, בריאות, חינוך, תחבורה, תקציב ועוד - המערכת מכירה מונחים ועושה היקשים לוגים וסימנטים בעברית וגם באנגלית

## 📄 מערכת עיבוד מסמכים חכם (Docling)

הוספתי למערכת מנוע עיבוד מסמכים מתקדם כזה שמאפשר למודל לקרוא ולהבין מסמכים בכל פורמט. עכשיו אפשר להעלות קובץ PDF, Word, Excel או תמונה סרוקה - והמודל יקרא אותם כמו בן אדם!

### מה מערכת הטיפול בקבצים יודעת לעשות?

המערכת בעצם רותמת למשימה הזו ״צבא קטן״ של סוכני בינה מלאכותית שרץ ברקע. מעין גירסאות דיגטליות של ״יועצים״ ו״מבקרי איכות״ שיש להם סט כלים אלגוריתמים ופקודות וגבולות גזרה שניתנו להם לשם השגת תוצאות ברות משמעות בתהליך נתוח מסמכים.

**והתוצאה :**

* 📑 **המרת מסמכים**: PDF, Word, Excel, PowerPoint, HTML - הכל הופך לטקסט מובנה שהמודל מבין

* 📊 **חילוץ טבלאות אוטומטי**: יש לכם PDF עם טבלאות? המערכת תחלץ אותן ותמיר אותן ל-JSON, CSV או Markdown

* 🖼️ **חילוץ תמונות**: המערכת מזהה תמונות במסמכים ומסווגת אותן (גרפים, דיאגרמות, תצלומים)

* 👁️ **OCR בעברית ובאנגלית**: סרקתם מסמך ישן? המערכת תזהה את הטקסט - גם בעברית! תומך גם בערבית ובשילובים

* 🚀 **עיבוד מקבילי**: מסמכים גדולים מעובדים ברקע עם התקדמות בזמן אמת

* 💾 **מטמון חכם**: מסמך שכבר עיבדתם? התוצאה נשמרת 24 שעות - חוסכים זמן וכוח עיבוד

### שלב 1: הורדת הפרויקט והגדרת סביבה 🛠️

לפני שנוריד את המודל והמערכת, עלינו להוריד את קבצי הפרויקט למחשב ולהגדיר את הסביבה.

1.  **הורדת הפרויקט (Clone)**:
    פתחו את הטרמינל (שורת הפקודה) והעתיקו את הפקודה הבאה:
    ```bash
    git clone https://github.com/oznav2/DictaChat.git
    cd BricksLLM
    ```

2.  **הגדרת קובץ משתני הסביבה (.env)**:
    המערכת צריכה קובץ הגדרות מיוחד כדי לדעת סיסמאות לשרתי פונקציות ומיקומי קבצים חשובים. הכנו עבורכם תבנית מוכנה בעברית.
    
    הריצו את הפקודה הבאה כדי ליצור את הקובץ:
    ```bash
    cp .env.heb .env
    ```

    **חשוב מאוד!** ⚠️
    עליכם לפתוח את הקובץ `.env` שנוצר ולשנות בו את הסיסמאות והמפתחות.
    חפשו שורות המכילות `your_..._here` והחליפו אותן בסיסמאות אמיתיות משלכם.
    
    לדוגמה, שנו את:
    `POSTGRESQL_PASSWORD=your_secure_postgres_password_here`
    למשהו כמו:
    `POSTGRESQL_PASSWORD=MySecretPassword123`

גם התלמיד או הסטודנט בפריפריה, כמו גם בעלת העסק הנאבקת על זכות קיומו של מקור פרנסתה, ובעיקר כל מי שבעצם אין לו הידע או האמצעים לגשר על הפערים הדרושים לרתום לעצמו ולטובתו בינה מלאכותית ישראלית חינמית וחכמה, יוכל מעכשיו להתנסות בשימוש במערכת בצורה חופשית ומוכנה לעבודה בסביבה מאובטחת.

זהו תהליך, וזו רק נקודת ההתחלה שלו. אני משוכנע שאם רק נרצה מספיק נוכל להנגיש אותו ליותר אנשים ובהרבה יותר פשטות ובקלות.

בהצלחה! ותהנו
