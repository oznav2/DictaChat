# אלגוריתם זיהוי תוצאות

   כדי לזהות תוצאות מתוך שיחה בשפה טבעית דיקטא משתמשת ב:

- **אותות חיוביים:** "זה עבד!", "מושלם", "בדיוק מה שהייתי צריך"
- **אותות שליליים:** "לא עבד", "זה נכשל", "לא עוזר"
- **אותות מרומזים:** שינויי נושא, נטישת גישה, שאילת שאלה דומה

```בקוד זה נראה ככה
# LLM מנתח את השיחה ומזהה תוצאה
if outcome == "worked":
    score += 0.2  # תוצאה חיובית
elif outcome == "failed":
    score -= 0.3  # תוצאה שלילית
elif outcome == "partial":
    score += 0.05  # הצלחה חלקית
```

# דיקטאצ'אט בעצם לומדת את הדפוסים שלכם
דיקטאצאט בונה 3 גרפי ידע מחוברים שלומדים היבטים שונים של הזיכרון שלכם:
באנגלית קוראים לזה : (Knowledge Graphs) 

### 1. גרף ניתוב (Routing KG) - "איפה לחפש"
לומד אילו אוספי זיכרון עונים על אילו סוגי שאלות.
- **אתם שואלים:** "אילו ספרים קראתי על השקעות?"
- **המערכת למדה:** שאלות על "ספרים" ← אוסף ספרים (85% הצלחה).
- **תוצאה:** מחפשת קודם בספרים, מדלגת על אוספים פחות רלוונטיים ← תשובות מהירות יותר.

### 2. גרף תוכן (Content KG) - "מה מחובר"
מחלץ אנשים, מקומות, כלים ומושגים מהזיכרונות שלכם ועוקב אחר מערכות יחסים.
- **הזיכרונות שלכם מציינים:** "שרה חן עובדת ב-אינטל כמהנדסת".
- **המערכת מחלצת:** שרה חן ← אינטל ← מהנדסת (עם ציוני איכות).
- **תוצאה:** "מי עובד ב-אינטל?" ← מוצאת את שרה גם ללא התאמה מדויקת של מילות מפתח.

### 3. גרף פעולה-מועילות (Action-Effectiveness KG) - "מה עובד"
עוקב אחר אילו כלים ופעולות מצליחים או נכשלים בהקשרים שונים.
- **בזמן חידון:** הבינה המלאכותית משתמשת ב-`create_memory()` כדי לענות על שאלות.
- **המערכת עוקבת:** `create_memory` ב-"recall_test" ← 5% הצלחה (הזיות!).
- **המערכת עוקבת:** `search_memory` ב-"recall_test" ← 85% הצלחה.
- **תוצאה:** לאחר 3+ כישלונות, מזהירה את הבינה המלאכותית: "`create_memory` בעל 5% הצלחה כאן - השתמש ב-`search_memory` במקום".

**ביחד:** גרף הניתוח מוצא את המקום הנכון, גרף התוכן מוצא מידע מחובר, וגרף הפעולה-מועילות מונע טעויות. שלושתם לומדים מתוצאות אמיתיות.
