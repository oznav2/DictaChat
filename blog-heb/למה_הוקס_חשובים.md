# למה הוקס (Hooks) חשובים?

כלי זיכרון אחרים מחכים שתשאלו. Roampal מזריקה הקשר באופן אוטומטי — לפני כל הנחיה (prompt) ואחרי כל תגובה. ללא קריאות ידניות. ללא שינויים בתהליך העבודה.

---

### לפני ההנחיה שלכם
ההוק `UserPromptSubmit` מזריק אוטומטית הקשר רלוונטי. Claude רואה מה עבד בעבר.

### אחרי כל תגובה
ההוק `Stop` לוכד את חילופי הדברים. הדירוג נאכף — הוא אינו אופציונלי.

---

## 7 כלי זיכרון
- `get_context_insights`
- `search_memory`
- `add_to_memory_bank`
- `update_memory`
- `delete_memory`
- `record_response`
- `score_response`

`score_response` הוא ייחודי ל-Claude Code — הוקס אוכפים דירוג בכל חילופי דברים.

---

## Claude Code מול Desktop MCP

| מאפיין | Claude Code | Desktop MCP |
| :--- | :--- | :--- |
| הזרקת הקשר | אוטומטית (הוקס) | ידנית (הנחיית LLM) |
| דירוג תוצאות | נאכף | בחירה (Opt-in) |
| למידה | כל חילופי דברים | כשאתם זוכרים לדרג |
| הכי טוב עבור | תהליך עבודה ללא חיכוך | משתמשים כבדים בריבוי כלים |

---

### משתמשים ב-Desktop MCP? טיפים לתוצאות טובות יותר:
- **הוסיפו להנחיית המערכת:** "בדוק ב-`search_memory` לקבלת הקשר לפני מתן תשובה".
- **הזכירו באמצע השיחה:** "בדוק בזיכרון לגבי מה שדנו בו בנוגע ל-X".
- **תעדו תוצאות:** "תעד ב-Roampal - זה עבד" או "...נכשל".

עם הוקס של Claude Code, זה קורה אוטומטית — ההוקס מזריקים הקשר כך שהבינה המלאכותית יודעת להשתמש בכלי הזיכרון.
